# 特定技能2号移行ナビ — PPT生成仕様書（AI入力用）

> このファイルは、PowerPoint初稿を生成AIに作らせるための**入力仕様書**です。このファイル1枚を渡せば、どの生成AI（ChatGPT／Claude／Gemini／Gamma／Canva等、種類を問わない）でも一定品質の初稿を作れる状態を目指しています。
>
> 内容のベースは `docs/sales/tokutei_navi_presentation_meeting.md`（初回面談専用・対話資料版）です。話法やトーンに迷った場合は、そちらのファイルの記載を優先してください。
>
> 本仕様書自体はまだPowerPointを作成するものではありません。生成AIへの**入力テンプレート**として使用してください。

---

## ① 資料目的

- 日本企業との**初回面談**で使用する資料である
- **営業資料ではなく、対話資料**として作成する
- 「共同検証」「共同棚卸し」を提案する資料である。契約やPoCの即断を迫るものではない
- ゴールは、企業担当者に「まず一緒に現状を整理してみませんか」と自然に感じてもらうことであり、その場での意思決定を求めることではない

---

## ② 想定読者

| 読者 | 関心事の傾向 |
|---|---|
| 経営者 | 投資対効果、リスク、意思決定に必要な情報量 |
| 人事責任者 | 実務運用、社内展開のしやすさ、情報管理 |
| 外国人材担当者 | 現場での運用負担、対象者一人ひとりの状況 |

いずれの読者に対しても、断定的な結論を急がず、状況整理の材料を提示する姿勢を貫くこと。

---

## ③ デザイン方針

| 方針 | 内容 |
|---|---|
| 全体トーン | シンプル、余白重視 |
| ビジュアル | アイコン中心。写真・イラストの多用は避ける |
| 配色 | 青・白を基調とする。過度な装飾色・グラデーションは使わない |
| 文字量 | 少なめ。1スライドの本文は目安3〜5行以内 |
| 構成原則 | 1スライド1メッセージを徹底する。1枚に複数の論点を詰め込まない |

## ④ スライドごとの目的・内容（Slide1〜Slide8）

### Slide1：タイトル・面談目的

| 項目 | 内容 |
|---|---|
| 目的 | 今日の面談が「判断の場」ではなく「対話の場」であることを最初の数秒で伝える |
| 載せる内容 | プロダクト名「特定技能2号移行ナビ」／サブタイトル「外食業法人向け　外国人材の2号移行準備状況を可視化する初期診断ツール」／面談目的（今日共有したいことを一言で） |
| 避ける内容 | 機能一覧、料金、契約に関する言葉、煽り文句 |
| 強調ポイント | 「対話である」ことが伝わるトーン。余白を大きく取り、文字数を最小限にする |

### Slide2：企業が抱える課題

| 項目 | 内容 |
|---|---|
| 目的 | 自社固有ではなく、業界共通の構造的課題であると認識してもらう |
| 載せる内容 | 分散管理されがちな4要素（在留期限／日本語レベル／技能試験の受験状況／管理者相当の実務経験）、その結果「誰が2号候補か見えにくい」という状態 |
| 避ける内容 | 「御社は対応が遅れている」といった否定的な断定、具体的な数値の決めつけ |
| 強調ポイント | 問いかけで終わる構成にする。スライド内の文言も断定せず、相手が話しやすい余地を残す |

### Slide3：共同検証（PoC）の考え方

| 項目 | 内容 |
|---|---|
| 目的 | 「PoC」という言葉のハードルを下げ、「共同検証」「共同棚卸し」という対話的な言い方に置き換える |
| 載せる内容 | 「共同検証」「共同棚卸し」という表現、5〜20名程度から始める理由、初期診断であり在留資格変更を保証するものではない旨、専門家確認へ接続する位置づけ |
| 避ける内容 | 「導入」「契約」「今すぐ」等の急かす表現、移行可否についての断定的な言及 |
| 強調ポイント | 「初期診断であり保証しない」の一文は、必ず視認できる大きさ・位置に配置する |

### Slide4：特定技能2号移行ナビ — 入力と可視化

| 項目 | 内容 |
|---|---|
| 目的 | 入力の手軽さと、匿名運用が可能であることを伝える |
| 載せる内容 | 入力：在留期限／日本語レベル／管理者相当の実務経験（必須3項目）、氏名または社員番号（任意）／可視化される内容：現在地と残り期間・3要件の充足状況・移行タイムライン・投資対効果・次にすべきこと |
| 避ける内容 | 実際のコード・計算ロジックの詳細説明、実物と異なる入力フォームの創作 |
| 強調ポイント | 「3つの入力だけで整理される」というシンプルさ |

### Slide5：診断結果イメージ

| 項目 | 内容 |
|---|---|
| 目的 | 実際の見え方を疑似体験してもらい、活用イメージを持ってもらう |
| 載せる内容 | 総合判定（○/△/✕）、現在地と残り期間、移行タイムライン、今すぐ取るべきアクション |
| 避ける内容 | 実在しない判定結果・数値の創作、「移行できます」等の断定表現 |
| 強調ポイント | 「初期診断であり最終判断ではない」旨を必ず明記する |

### Slide6：企業へ返却する成果物

| 項目 | 内容 |
|---|---|
| 目的 | 共同検証への協力が、具体的で持ち帰れる成果物につながることを示す |
| 載せる内容 | 候補者一覧／不足要件一覧／優先順位／棚卸しレポート／改善提案の5点 |
| 避ける内容 | これらをツールのスクリーンショットとして表現すること（実際には共同検証後に作成する別紙資料であり、ツール画面には存在しない） |
| 強調ポイント | 「画面ではなく、結果を集計した別紙資料として提供される」ことを明記する。資料全体の中で最も具体的な価値提示の場であるため、他スライドより丁寧に扱う |

### Slide7：関連プロダクト（支援ロードマップ）

| 項目 | 内容 |
|---|---|
| 目的 | 今日の話が全体像の一部であり、無理に全部を一度に導入する必要がないことを伝える |
| 載せる内容 | Step1 特定技能2号移行ナビ（移行タイムラインの可視化）→ Step2 PreCheck（日本語能力の初期現在地確認）→ Step3 インテグリティ診断（管理職素養・投資継続可否の参考情報） |
| 避ける内容 | インテグリティ診断を「採用可否の判定」であるかのように書くこと。正しくは「投資継続可否・管理職素養確認の参考情報」であり、採用選別のためのものではない |
| 強調ポイント | 「必要に応じて選べる」選択肢としての位置づけ |

### Slide8：次のアクション

| 項目 | 内容 |
|---|---|
| 目的 | 「今日は決断の場ではない」という安心感を残しつつ、次のステップを明確にする |
| 載せる内容 | 面談後 → 対象者選定 → 共同検証 → レビュー → （必要なら）正式導入検討、という流れ |
| 避ける内容 | 「今すぐ契約」「今日ご決断を」等のクロージングを迫る表現 |
| 強調ポイント | 「正式導入検討」は選択肢の一つであり、必須のゴールではないという留保を明記する |

| 区分 | スライド | 内容 |
|---|---|---|
| 必須 | Slide4 | 入力フォーム画面全体（氏名/社員番号欄〜診断ボタンまで） |

画像を配置する箇所では、生成AIが実際のスクリーンショットを創作するのではなく、「ここに実際のLovable画面のスクリーンショットを挿入」という**プレースホルダー枠**を配置すること。

---

## ⑥ AIへの禁止事項

1. **制度判断をしない**：在留資格変更の可否を判定・示唆する記述を追加しない
2. **断定しない**：「必ず」「確実に」「保証する」「間違いなく」等の断定表現を使わない
3. **画像を創作しない**：実在しないスクリーンショットや画面イメージを生成しない。画像が必要な箇所はプレースホルダーのみとする
4. **統計を創作しない**：実在しない数値・調査データ・パーセンテージ・利用実績等を作らない
5. **画面に存在しないUIを書かない**：実際のLovable画面にない項目・ボタン・機能をスライド内で図示・言及しない
6. **リンク先URLを創作しない**：本資料の内容に登場するURL（PreCheck、インテグリティ診断等）は、実在の接続先以外を新たに作らない

## ⑦ 出力条件

- PowerPoint（またはスライド資料）として成立する構成にすること
- 文字量は少なめにすること（1スライドの本文は目安3〜5行以内）
- 日本企業向けの丁寧なビジネス日本語で作成すること
- 営業色を強くしないこと（煽り文句・即決を迫る表現を避ける）
- デザインより先に、読みやすさ・伝わりやすさを優先すること
