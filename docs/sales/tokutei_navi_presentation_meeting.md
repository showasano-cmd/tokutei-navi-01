# 特定技能2号移行ナビ — 初回面談用対話資料

> このファイルは、日本企業との**初回面談専用**に作成した対話資料です。`docs/sales/tokutei_navi_presentation.md`（標準版・プロダクト紹介中心）、`docs/sales/tokutei_navi_presentation_poc_first.md`（PoC版・PoC提案中心）とは目的が異なり、この2ファイルは変更せずそのまま保存されています。
>
> **この資料の目的は「ツール紹介」でも「PoC契約」でもありません。** 企業担当者に「まず一緒に現状を整理してみませんか」と自然に感じてもらうことです。営業色を抑え、コンサルティング型の対話資料として設計しています。
>
> 全8枚構成。各スライドは「タイトル／掲載内容／話す内容／表示するLovable画面／企業側へ伝えたいメッセージ」の5項目で記述しています。
>
> 中心語彙は「可視化」「初期診断」「棚卸し」「共同検証」「共同棚卸し」「専門家確認」とし、「PoC」という言葉は使う場面を限定します（Slide3で一度、概念説明として使用）。断定表現・制度判断は行いません。

---

## 全体構成

```
Slide1 タイトル・面談目的
   ↓
Slide2 企業が抱える課題（問いかけで終える）
   ↓
Slide3 共同検証（PoC）の考え方
   ↓
Slide4 特定技能2号移行ナビ：入力と可視化
   ↓
Slide5 診断結果イメージ
   ↓
Slide6 企業へ返却する成果物
   ↓
Slide7 関連プロダクト（支援ロードマップ）
   ↓
Slide8 次のアクション
```

この面談のゴールは、契約でもPoCの正式合意でもなく、**「一度、一緒に現状を整理してみましょう」という空気を作ること**です。数字や機能の説明よりも、相手が話しやすい間合いを作ることを優先します。

## Slide1：タイトル・面談目的

| 項目 | 内容 |
|---|---|
| タイトル | 特定技能2号移行ナビ |
| 掲載内容 | サブタイトル「外食業法人向け　外国人材の2号移行準備状況を可視化する初期診断ツール」／面談目的「今日の面談で共有したいこと」として、①現状の可視化の考え方②一緒に確認できることの2点のみを簡潔に提示 |
| 話す内容 | 「本日はお時間をいただきありがとうございます。今日は何かをご提案してその場でご判断いただく場ではなく、貴社の特定技能1号人材の状況を一緒に確認させていただく時間にできればと思っています」 |

| 企業側へ伝えたいメッセージ | 今日は「売り込みの場」ではなく「一緒に状況を確認する場」であるという前提をそろえる。冒頭でこの空気を作ることが本資料全体で最も重要な部分 |

## Slide2：企業が抱える課題

| 項目 | 内容 |
|---|---|
| タイトル | 在留期限・日本語・技能試験・実務経験は、それぞれ別に管理されていませんか |
| 掲載内容 | 4つの情報（在留期限／日本語レベル／技能試験の受験状況／管理者相当の実務経験）が分散管理されがちな実態を提示。結果として「誰が2号移行の候補になり得るか」が見えにくい状態になっていることを示す |
| 話す内容 | 「多くの企業様で、在留期限は管理表に記録されていても、日本語レベル・技能試験の受験状況・管理者相当の実務経験はそれぞれ別に管理されていることが多いようです。結果として『誰が2号移行の候補になり得るか』を一覧で見える状態にするのが難しい、という声をよく伺います。御社ではいかがでしょうか」（相手への問いかけで締め、一方的な説明で終わらせない） |
| 表示するLovable画面 | なし |
| 企業側へ伝えたいメッセージ | 自社固有の問題ではなく、業界に共通する構造的な状態であると感じてもらい、防御的にならず率直に話しやすい空気を作る |

## Slide3：共同検証（PoC）の考え方

| 項目 | 内容 |
|---|---|
| タイトル | まずは一緒に、5〜20名分だけ棚卸ししてみませんか |
| 掲載内容 | 「共同検証」「共同棚卸し」という言葉で、一般的な「PoC」の概念を言い換えて説明。5〜20名程度から始める理由（大きな負担をかけずにまず感触を掴んでもらうため）。**診断結果は初期診断であり、在留資格変更を保証するものではないこと**。結果は専門家（行政書士・登録支援機関等）への確認につなげるための材料として位置づけること、の3点を明記 |
| 話す内容 | 「ここでいう『PoC』は、契約や導入を前提にしたものではなく、貴社と私たちで一緒に、5〜20名程度の情報をもとに現状を棚卸ししてみる、という意味での共同検証です。人数を絞るのは、大きな負担をかけずにまず感触を掴んでいただきたいからです。ここで出てくる結果はあくまで初期診断であり、在留資格変更を保証するものではありません。むしろ、専門家に相談すべき人材を早めに見極めるための材料として使っていただくことを想定しています」 |
| 表示するLovable画面 | なし（この枚は概念説明に専念し、画面を見せると『ツールの話』に意識が戻ってしまうため、あえて画面を使わない） |
| 企業側へ伝えたいメッセージ | 「PoC」という言葉が持つ「契約」「導入」のニュアンスを外し、「小さく一緒にやってみる」共同作業として捉え直してもらう。心理的ハードルを下げることが目的 |

## Slide4：特定技能2号移行ナビ — 何を入力し、何が可視化されるか

| 項目 | 内容 |
|---|---|
| タイトル | 3つの情報を入れると、状況がその場で整理されます |
| 掲載内容 | 入力：①在留期限　②日本語レベル　③管理者相当の実務経験　④氏名または社員番号（任意）／可視化される内容：①現在地と残り期間　②3要件の充足状況　③移行タイムライン　④投資対効果　⑤次にすべきこと |
| 話す内容 | 「実際に使っているツールをご覧いただきながらご説明します。入力していただくのは在留期限、日本語レベル、管理者相当の実務経験の3つだけです。氏名の代わりに社員番号を使っていただくこともできるので、匿名の状態で確認いただくことも可能です」 |
| 表示するLovable画面 | 入力フォーム画面全体（氏名/社員番号欄〜診断ボタンまで） |
| 企業側へ伝えたいメッセージ | 操作の負担が小さいこと、匿名でも運用できることを実感してもらう。「これなら試せそう」という感覚を持ってもらう |

## Slide5：診断結果イメージ

| 項目 | 内容 |
|---|---|
| タイトル | 入力すると、こう見えてきます |
| 掲載内容 | 実際の診断結果画面（総合判定・現在地・移行タイムライン）。何が分かるか（誰が・いつまでに・何が不足しているか）。何が次の行動になるか（今すぐ取るべきアクション） |
| 話す内容 | 「入力するとその場でこのように結果が出ます。○/△/✕は在留期限までの3条件の状況を大まかに示す目安で、その下にN3の受験機会や技能試験の受験機会が時系列で並びます。これを見ながら、次に何をすべきかまで一緒に確認いただけます。繰り返しになりますが、これは初期診断であり、最終的な判断ではありません」 |
| 表示するLovable画面 | 診断結果カード（総合判定ヘッダー＋「01 現在地と残り期間」＋「03 移行タイムライン」＋「今すぐ取るべきアクション」） |
| 企業側へ伝えたいメッセージ | 実際の見え方を体験してもらい、「これなら社内の状況把握に使えそうだ」という実感を持ってもらう |

## Slide6：企業へ返却する成果物

| 項目 | 内容 |
|---|---|
| タイトル | 共同検証を終えると、こちらをお渡しします |
| 掲載内容 | ①候補者一覧（対象者ごとの総合判定・状況の一覧）　②不足要件一覧（誰の・どの要件が・どれだけ不足しているか）　③優先順位（対応を急ぐべき人材の順位付け）　④棚卸しレポート（専門家確認が必要な人材／学習支援が必要な人材等の区分整理）　⑤改善提案（辞令整備・学習計画着手等、次に取るべき行動の提案） |
| 話す内容 | 「共同検証が終わった後には、対象者ごとの結果を一覧にした候補者一覧、どの要件がどれだけ不足しているかをまとめた不足要件一覧、対応の優先順位、専門家確認が必要な人材を整理した棚卸しレポート、そして次に取るべきアクションの改善提案をお渡しします。これらはツールの画面そのものではなく、結果を持ち帰って整理した別紙としてご用意するものです」 |
| 表示するLovable画面 | なし。**候補者一覧・不足要件一覧・優先順位・棚卸しレポート・改善提案は、いずれもツールの画面上に存在する機能ではなく、共同検証終了後に個別の診断結果を集計・整理してお渡しする別紙資料である**。画面が存在する前提で誤解されないよう、この枚ではスクリーンショットを使用しない。イメージ図・簡易モックアップを別途用意する場合は、それが「別紙のイメージ」であることをキャプションで明記する |
| 企業側へ伝えたいメッセージ | 共同検証への協力が、具体的で持ち帰れる成果物につながることを実感してもらう。「情報提供して終わり」ではなく、「整理された状態で返ってくる」という価値を明確に伝える。このスライドは資料全体の中で最も具体的な価値提示の場であるため、他のスライドより時間をかけて説明する |

## Slide7：関連プロダクト（支援ロードマップ）

| 項目 | 内容 |
|---|---|
| タイトル | 支援のロードマップ |
| 掲載内容 | Step1 特定技能2号移行ナビ（移行タイムラインの可視化）→ Step2 PreCheck（日本語能力の初期現在地確認）→ Step3 インテグリティ診断（管理職素養・投資継続可否の参考情報）。単なる製品紹介ではなく、「今日必ず使うもの」ではなく「必要に応じて次に進める選択肢」としてのロードマップとして整理する |
| 話す内容 | 「今日ご説明しているのは、このロードマップの最初のステップです。まず移行タイムラインを一緒に可視化し、必要に応じて日本語能力の現在地を確認するPreCheck、管理職素養や投資継続可否を考える材料としてのインテグリティ診断へと進めることができます。すべてを今日決めていただく必要はなく、状況に応じて選んでいただけます」 |
| 表示するLovable画面 | 診断結果下部のCTAセクション（PreCheck・インテグリティ診断への導線カード）。使う場合は控えめに、全体像を示す図の補足として使用する程度にとどめる |
| 企業側へ伝えたいメッセージ | 「今日の話は全体像のごく一部であり、無理に全部を一度に導入する必要はない」という安心感を与えつつ、将来的な広がりがあることを示す |

## Slide8：次のアクション

| 項目 | 内容 |
|---|---|
| タイトル | 今日からのステップ |
| 掲載内容 | 「面談後 → 対象者選定 → 共同検証 → レビュー → （必要なら）正式導入検討」という流れを図示。「正式導入検討」はあくまで選択肢の一つであり、必須の到達点ではないことを明記 |
| 話す内容 | 「今日お話しした内容を踏まえて、まずは対象者を一緒に選び、5〜20名程度で共同検証を行い、その結果を一緒にレビューする、という流れを考えています。そこから先、正式に広げるかどうかは、レビューの結果を見てから判断いただければと思います。今日この場で何かを決めていただく必要はありません」 |
| 表示するLovable画面 | なし |
| 企業側へ伝えたいメッセージ | 「今日は決断の場ではない」ことを最後にもう一度伝え、次に進むための心理的な負担を最小化する。次回に向けた自然な合意形成で締めくくる |
