# 特定技能2号移行ナビ — 初回面談 想定質問集

> 日本企業との初回面談で想定される質問と、回答の方向性を整理した資料です。経営者・人事責任者・外国人材担当者の3つの立場から、それぞれ10問程度、合計30問を目安にまとめています。
>
> **本資料は制度判断を行いません。** 在留資格変更の可否に関わる質問には、状況の可視化・整理の範囲で回答し、必要に応じて行政書士・登録支援機関等への「専門家確認」へ自然につなげてください。断定表現・誇張・保証も行いません。

---

## 経営者からの想定質問

### Q1. このツールを使うことで、実際に2号移行の合格率は上がるのですか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 投資対効果を重視する経営者は、成果に直結するかどうかをまず確認したい |
| 推奨回答 | 「合格率そのものを保証するものではありません。状況を可視化し、早い段階で対応に着手できるようにすることで、準備不足による見送りを減らせる可能性がある、とお伝えしています」 |
| 回答時の注意点 | 「上がります」と断定しない。数値・統計を創作しない。可視化による効果に絞って説明する |

### Q2. 費用はいくらかかりますか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 投資判断のため、費用感を早期に把握したい |
| 推奨回答 | 「本日ご提案しているのは、まず5〜20名規模の共同検証です。具体的な費用感は貴社の状況に応じて個別にご相談させてください」 |
| 回答時の注意点 | 資料に明記のない金額をその場で創作しない。持ち帰り事項として扱う |

### Q3. 導入した場合、社内のどの部署が運用を担うことになりますか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 運用体制・工数への懸念 |
| 推奨回答 | 「まずは人事・店舗管理のご担当者様が対象者の情報を入力いただく形を想定しています。詳細は共同検証を通じて一緒に確認できればと思います」 |
| 回答時の注意点 | 運用体制を断定的に決めつけず、企業ごとに異なる前提で話す |

### Q4. このツールを使わなかった場合、具体的にどんなリスクがありますか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | リスクの大きさを把握し、優先度を判断したい |
| 推奨回答 | 「ツールの利用有無に関わらず、在留期限の管理が分散していると対応の遅れに気づきにくい、という点は一般的に指摘されます。まずは現状を可視化すること自体に価値があると考えています」 |
| 回答時の注意点 | 「使わないと危険です」という煽りにならないよう、可視化そのものの価値に焦点を当てる |

### Q5. 他社にも同じような提案をしているのですか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 差別化・優先度の確認 |
| 推奨回答 | 「外食業を中心に、同様の課題をお持ちの企業様にご案内していますが、進め方は各社の状況に合わせて個別にご相談しています」 |
| 回答時の注意点 | 他社の実名・実績を誇張しない。特定企業名を出さない |

### Q6. このツールの提供元はどこですか？セキュリティは大丈夫ですか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 情報管理の安全性への懸念 |
| 推奨回答 | 「認証やデータベースを持たない、クライアントサイドで完結する仕組みで、入力情報はサーバーに保存されません。詳細な情報管理方針については別途ご説明します」 |
| 回答時の注意点 | 事実（README記載のバックエンドなし・クライアントサイド完結）に基づいて答える。誇張したセキュリティ主張をしない |

### Q7. 共同検証の後、契約しなければならないのですか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 「一度試したら後戻りできないのでは」という懸念 |
| 推奨回答 | 「共同検証自体に契約義務はありません。結果をご覧いただいたうえで、正式導入を検討するかどうかはあらためてご判断いただけます」 |
| 回答時の注意点 | 契約義務があるかのような誤解を与えない。次のアクションはあくまで任意であることを明言する |

### Q8. 経営会議でこの話をする場合、何を持ち帰ればよいですか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 社内説明のための材料を求めている |
| 推奨回答 | 「本日の資料に加え、ヒアリングシートの記入結果や共同検証の対象人数感を持ち帰っていただけると、社内でのご説明に使いやすいかと思います」 |
| 回答時の注意点 | 具体的な持ち帰り物を明確に案内する |

### Q9. このツールの診断結果はどの程度信頼できますか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 判断材料としての信頼性を確認したい |
| 推奨回答 | 「在留期限や試験スケジュール等の入力情報から機械的に算出する初期診断であり、個別の事情を網羅するものではありません。あくまで状況整理・優先順位付けのための材料としてご活用いただくものです」 |
| 回答時の注意点 | 「精度が高い」等の品質保証をしない。初期診断であることを繰り返し伝える |

### Q10. 競合の類似サービスと比べてどう違いますか？

| 項目 | 内容 |
|---|---|
| 質問の背景 | 比較検討をしたい |
| 推奨回答 | 「他社サービスの詳細については把握しかねますが、当ツールは在留期限・日本語レベル・実務経験の3条件を統合的にタイムラインとして可視化する点を特徴としています」 |
| 回答時の注意点 | 競合の悪口や、比較優位性の誇張をしない |

## 人事責任者からの想定質問

### Q11. 入力する情報は、誰が用意すればよいですか？

|---|---|
| 質問の背景 | 実務担当として準備の負荷を把握したい |

| 回答時の注意点 | 情報源を明確にしつつ、企業ごとに管理方法が異なることを踏まえて答える |

### Q12. 個人情報の取り扱いはどうなりますか？

| 項目 | 内容 |
|---|---|

| 回答時の注意点 | 事実に基づいた説明にとどめ、法的な保証はしない。詳細は貴社の情報管理ルールに従っていただくよう伝える |

| 回答時の注意点 | 誤入力時のリスクを過度に強調せず、再入力できる事実を淡々と伝える |

## 回答時の共通ルール

- 制度判断（在留資格変更の可否そのものの判定）は行わない。判断が必要な質問は、必ず「専門家確認」へ接続する形で答える
- 「必ず」「確実に」「保証します」といった断定表現は使わない
- 実際にない機能・実績・統計を、その場で創作して答えない
- 分からない・その場で答えられない質問は、無理に即答せず「持ち帰って確認します」と伝えてよい
- 面談全体の進め方は `docs/sales/tokutei_navi_meeting_flow.md`、各スライドの話し方は `docs/sales/tokutei_navi_presenter_notes.md` を併せて参照してください
