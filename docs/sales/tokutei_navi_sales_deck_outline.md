# 特定技能2号移行ナビ — 面談用スライド構成案（6〜8枚）

> 日本企業との面談で使用するスライドの構成案です。実際のスライド制作前の台本として使用してください。各スライドは「タイトル／要点／話すメッセージ／見せる画面」の4項目で構成しています。

---

## 全体の流れ

```
課題提起 → 現状の可視化 → プロダクト紹介 → デモ（画面提示） → ファネル全体像 → PoC提案 → 次のアクション
```

面談の目的は「その場で契約を取る」ことではなく、**課題認識の一致とPoCへの合意**です。断定的な表現（「必ず移行できる」「許可される」）は使わず、「可視化」「棚卸し」「初期診断」を軸に説明します。

---

## スライド1：タイトル・課題提起

| 項目 | 内容 |
|---|---|
| タイトル | 特定技能1号人材、在留期限までに2号移行は間に合っていますか？ |
| 要点 | コロナ禍以降に入国した特定技能1号人材の在留期限が2025〜2026年に集中到来している |
| 話すメッセージ | 「在留期限は管理表で把握していても、N3・技能試験・実務経験の3条件が同時進行で必要という点まで整理できている企業は多くありません」という問題提起から入る |
| 見せる画面 | なし（口頭のみ、または本資料の表紙） |

---

## スライド2：現状の課題

| 項目 | 内容 |
|---|---|
| タイトル | なぜ移行タイムラインの管理が難しいのか |
| 要点 | 誰が・いつ・何をすべきかの管理が属人化している／3条件が同時進行で必要／代替ルート（80%特例）の検討が後手になりがち |
| 話すメッセージ | 「担当者の頭の中にしかない」状態のリスクを具体的に指摘する。人事異動・退職時に情報が失われるリスクにも触れる |
| 見せる画面 | なし（口頭・箇条書きスライド） |

---

## スライド3：プロダクト紹介

| 項目 | 内容 |
|---|---|
| タイトル | 特定技能2号移行ナビとは |
| 要点 | 3項目の入力だけで、対象人材ごとの移行タイムラインを初期診断として可視化するツール |
| 話すメッセージ | 「在留期限・日本語レベル・実務経験開始日の3つを入れるだけです」とシンプルさを強調。入力のハードルの低さがPoCへの合意を得やすくする |
| 見せる画面 | トップの入力フォーム画面（hero〜フォーム部分） |

---

## スライド4：デモ（入力〜診断結果）

| 項目 | 内容 |
|---|---|
| タイトル | 実際の診断結果を見てみる |
| 要点 | 総合判定（○/△/✕）、現在地と残り期間、3要件チェック、移行タイムラインを画面上で見せる |
| 話すメッセージ | 実データではなく仮の日付でその場入力し、結果が即時に出ることを体験してもらう。「これはあくまで初期診断であり、申請可否を保証するものではない」ことをこのタイミングで明確に伝える |
| 見せる画面 | 診断結果カード（総合判定〜移行タイムラインのセクション） |

---

## スライド5：投資対効果・アクション提案

| 項目 | 内容 |
|---|---|

| 項目 | 内容 |

## 補足：面談時の注意事項

- 診断ロジック・判定基準・リンクURLなど、ツール自体の仕様には言及するが、その場での変更依頼や機能追加の即答は避け、持ち帰りとする。
- 「必ず移行できる」「許可される」といった断定的な表現は使用しない。常に「初期診断」「可視化」「専門家確認が必要」という前提を明示する。
- 個人が特定される実データはデモに使用しない。仮の日付・仮名で操作する。
