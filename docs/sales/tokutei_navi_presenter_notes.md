# 特定技能2号移行ナビ — 発表者ノート（Slide1〜8）

> PowerPoint完成後、面談担当者が実際に話す際の補助資料です。読み上げ原稿ではなく、**話者が自分の言葉で話せるようにするための発表者ノート**として作成しています。
>
> 対象は `docs/sales/tokutei_navi_presentation_meeting.md`（初回面談用対話資料）に基づくSlide1〜8です。PowerPoint・Markdown本体（Single Source of Truth）はいずれも変更していません。
>
> 全体を通じて、「共同検証」「共同棚卸し」「初期診断」「専門家確認」という言葉を軸に話すことを意識してください。制度判断・断定表現・誇張・保証は行いません。

---

## Slide1：タイトル・面談目的

| 項目 | 内容 |
|---|---|
| スライドの目的 | 面談全体のトーンを決定づける。「今日は判断を迫る場ではない」という空気を最初に作る |
| このスライドで絶対伝えること | 今日は情報を一方的に提供する場でも、その場で決めていただく場でもなく、一緒に状況を確認する時間であること |
| 話すポイント（60〜90秒） | 挨拶→本日の趣旨（一緒に状況を確認する時間であること）→簡単な自己紹介とツールの一言紹介→「今日は何かを決めていただく場ではありません」と明言する |
| 強調する一文 | 「今日は判断をお願いする場ではなく、まず状況を一緒に確認させてください」 |
| 言わない方がよいこと | 「導入すれば」「契約すれば」といった前のめりな表現、価格の話、競合他社との比較 |
| 次スライドへの自然なつなぎ方 | 「まず、多くの企業様でよく伺う状況からお話しさせてください」と課題のスライドへ移る |

## Slide2：企業が抱える課題

| 項目 | 内容 |
|---|---|
| スライドの目的 | 自社固有ではなく業界共通の構造的課題であると認識してもらい、相手に話してもらう入口を作る |
| このスライドで絶対伝えること | 在留期限・日本語レベル・技能試験・管理者相当経験がそれぞれ別々に管理されがちであること |
| 話すポイント（60〜90秒） | 4つの管理項目を一つずつ挙げる→結果として「誰が2号候補か見えにくい」という状態を提示する→「御社ではいかがでしょうか」と問いかけ、相手の反応を待つ（間を作ることを恐れない） |
| 強調する一文 | 「誰が2号移行の候補になり得るか、一覧で見える状態になっていますか」 |
| 言わない方がよいこと | 「御社は管理が甘い」といった否定的な評価、決めつけの言い回し |
| 次スライドへの自然なつなぎ方 | 相手の回答を受け止めたうえで「そうした状況を、まず一緒に整理する方法があります」と共同検証のスライドへ移る |

## Slide3：共同検証（PoC）の考え方

| 項目 | 内容 |
|---|---|
| スライドの目的 | 「PoC」という重く聞こえる言葉のイメージを外し、「一緒に小さくやってみる」共同作業として捉え直してもらう |
| このスライドで絶対伝えること | 共同検証は契約前提ではないこと、5〜20名程度の小規模から始める理由、診断結果は初期診断であり在留資格変更を保証しないこと、結果は専門家確認へつなげる位置づけであること |
| 話すポイント（60〜90秒） | 「共同検証」という言葉の意味を説明→なぜ5〜20名なのか（負担を抑え、まず感触を掴んでもらうため）→結果はあくまで初期診断であることを明言→専門家確認への接続を明示する |
| 強調する一文 | 「これは契約のご相談ではなく、まず一緒に現状を棚卸ししてみる、というご提案です」 |
| 言わない方がよいこと | 「今すぐ」「今日中に」といった急かす表現、「必ず移行できます」といった断定 |
| 次スライドへの自然なつなぎ方 | 「実際にどんなツールを使って棚卸しするのか、少し画面を見ながらご説明します」と入力・可視化のスライドへ移る |

## Slide4：特定技能2号移行ナビ — 入力と可視化

| 項目 | 内容 |
|---|---|
| スライドの目的 | 操作の手軽さと、匿名運用が可能であることを実感してもらう |
| このスライドで絶対伝えること | 入力は3項目のみで完結すること、氏名の代わりに社員番号を使えば匿名運用ができること |
| 話すポイント（60〜90秒） | 画面を見せながら在留期限・日本語レベル・管理者相当経験の3項目を説明→氏名/社員番号欄が任意であることに触れる→可視化される5要素（現在地/3要件/タイムライン/投資対効果/次アクション）を簡潔に列挙する |
| 強調する一文 | 「入力は3つだけで、氏名の代わりに社員番号でもご利用いただけます」 |
| 言わない方がよいこと | 内部の計算ロジックや開発の裏側の詳細説明（相手の関心事ではなく、時間を消費するだけになりやすい） |
| 次スライドへの自然なつなぎ方 | 「実際に入力すると、こんな風に結果が見えてきます」と診断結果イメージのスライドへ移る |

## Slide5：診断結果イメージ

| 項目 | 内容 |
|---|---|
| スライドの目的 | 実際の見え方を体験してもらい、活用イメージを持ってもらう |
| このスライドで絶対伝えること | 総合判定・タイムライン・次のアクションが一目で分かること、そしてこれが初期診断であり最終判断ではないこと |
| 話すポイント（60〜90秒） | 画面を見せながら○/△/✕の意味を説明→タイムラインの意味（JLPT・技能試験のスケジュール照合）に触れる→今すぐ取るべきアクション欄を紹介→「これは初期診断であり、最終的な判断ではありません」と必ず一度明言する |
| 強調する一文 | 「これはあくまで初期診断です。最終的な判断は、専門家の確認を経て行っていただくものです」 |
| 言わない方がよいこと | 「この判定なら移行できます」といった断定、個別の結果に対する断言 |
| 次スライドへの自然なつなぎ方 | 「この結果をもとに、共同検証の後にはこういったものをお渡しできます」と成果物のスライドへ移る |

## Slide6：企業へ返却する成果物

| 項目 | 内容 |
|---|---|
| スライドの目的 | 共同検証への協力が具体的なリターンにつながることを示す。資料全体の中で最も価値提示が強い場面であり、時間をかけて丁寧に話す |
| このスライドで絶対伝えること | 候補者一覧・不足要件一覧・優先順位・棚卸しレポート・改善提案の5点が返却されること、これらはツールの画面ではなく別紙資料であること |
| 話すポイント（60〜90秒） | 5つの成果物を一つずつ、なぜ役立つかを添えて紹介する→「これらはツールの画面そのものではなく、結果を集計してお渡しする別紙です」と明言する |
| 強調する一文 | 「情報をいただくだけでなく、整理された状態でお返しします」 |
| 言わない方がよいこと | 成果物をあたかも既存の画面機能であるかのように話すこと、成果物の完成度を実際以上に誇張すること |
| 次スライドへの自然なつなぎ方 | 「この先、必要に応じて広げられる選択肢もあります」と支援ロードマップのスライドへ移る |

## 全体を通じての心構え

- 各スライドの「話すポイント」は暗記して読み上げるものではなく、要点を掴んだうえで自分の言葉で話すための土台です
- 沈黙や間を恐れないこと。特にSlide2・Slide3の問いかけの後は、相手が考える時間を確保する
- 「初期診断」「保証しない」「専門家確認」という言葉は、資料上の記載だけに頼らず、口頭でも自分の言葉で必ず一度伝える
- 面談全体の進行については `docs/sales/tokutei_navi_meeting_flow.md` を、想定質問への回答準備には `docs/sales/tokutei_navi_expected_questions.md` を併せて参照してください
