|---|---|
| タイトル | なぜ移行タイムラインの管理が難しいのか |
| 掲載内容 | 企業で実際に起きている課題を3点で整理：①在留期限管理だけが独立して行われている　②N3・技能試験・管理者相当経験がそれぞれ別管理になっている　③結果として「誰が2号へ移行できるか」が見えない |

## Slide3：特定技能2号移行ナビとは

| 項目 | 内容 |
|---|---|
| タイトル | 3つの入力から移行タイムラインを可視化する |
| 掲載内容 | 入力：①在留期限　②日本語レベル　③管理者相当経験　④氏名または社員番号（任意）／出力：①総合判定　②残り期間　③タイムライン　④投資対効果　⑤次アクション |
| 話す内容 | 「入力は在留期限・日本語レベル・管理者相当経験の3つが必須です。氏名または社員番号は任意で、社員番号のみでも匿名運用が可能です。入力するとその場で、総合判定からタイムライン、投資対効果までを初期診断として可視化します」 |
| 表示するLovable画面 | 入力フォーム画面全体（氏名/社員番号欄〜在留期限・日本語レベル・管理者相当経験・投資コスト欄・診断ボタンまで） |
| 企業側に伝えたいメッセージ | 入力のハードルが低いこと、匿名運用ができること、出力が単なる合否判定ではなく複数の切り口で状況を可視化するものであることを伝える |

## PowerPoint化する際の注意点

- スクリーンショットは実データではなく、仮の日付・仮名（または「EMP001」等の社員番号）で取得すること。個人が特定される実データは使用しない。
- Slide4の診断結果スクリーンショットを単独で見せる場合も、「初期診断であり申請可否を保証しない」旨をスライド内のキャプション等で必ず併記する。画面外の口頭説明だけに頼らない。
- 判定色（○=緑系／△=黄系／✕=赤系）が分かるよう、スクリーンショットは十分な解像度で取得し、トリミング時に配色が潰れないようにする。
- 1スライド1メッセージを基本とし、表の情報をそのままスライドに詰め込みすぎない。「話す内容」を口頭説明に回し、スライド上の文字量は要点のみに絞る。
- 本ファイル（Markdown正本）を更新した場合は、PowerPoint／Google Slides側にも同じ内容を反映し、正本とスライドの内容がずれないよう運用する。
- 断定表現（「必ず」「保証する」「許可される」等）は、スライド化の過程でも追加しない。特にSlide4・5・6は営業的な誇張が入りやすいため、Markdown正本の文言から逸脱しないよう注意する。
