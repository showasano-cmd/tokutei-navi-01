# 特定技能2号移行ナビ — 初回面談 進行ガイド

> 日本企業との初回面談（30〜60分を想定）の進行ガイドです。営業マニュアルではなく、**対話を自然に進めるための支援ドキュメント**として作成しています。
>
> 全体を通じて「共同検証」「共同棚卸し」「初期診断」「専門家確認」という考え方を軸にし、制度判断・断定表現・誇張・保証は行いません。
>
> スライドごとの話し方は `docs/sales/tokutei_navi_presenter_notes.md`、想定質問への回答準備は `docs/sales/tokutei_navi_expected_questions.md`、ヒアリング項目の詳細は `docs/hearing/company_hearing_sheet.md` を併せて参照してください。

---

## 全体の時間配分の目安

| フェーズ | 時間配分（60分想定） | 時間配分（30分想定） |
|---|---|---|
| ① 面談開始 | 2〜3分 | 1〜2分 |
| ② アイスブレイク | 3〜5分 | 1〜2分（省略も可） |
| ③ 会社理解 | 5分 | 2〜3分 |
| ④ 課題確認 | 5分 | 3分 |
| ⑤ PowerPoint説明 | 15〜20分 | 10〜12分 |
| ⑥ ヒアリング | 5〜10分 | 3〜5分 |
| ⑦ 共同検証の提案 | 5分 | 3分 |
| ⑧ クロージング | 3〜5分 | 2分 |
| ⑨ 次回アクション | 2〜3分 | 2分 |

30分の面談では、アイスブレイクと会社理解を簡略化し、PowerPoint説明（特にSlide4のツール詳細説明）を圧縮することを優先してください。Slide3（共同検証の考え方）とSlide6（成果物）、および次回アクションの確認は、時間が短い場合でも省略しないことを推奨します。

---

## ①面談開始

| 項目 | 内容 |
|---|---|
| 目的 | 場の空気を作り、今日が判断の場ではないことを最初に共有する |
| 時間配分 | 2〜3分（60分想定）／1〜2分（30分想定） |
| 確認事項 | 出席者の役職・人数、今日のゴール認識のすり合わせ |
| 話すポイント | 挨拶、自己紹介、本日の趣旨（一緒に状況を確認する時間であること）を最初に伝える |
| 避ける話題 | 契約条件、価格の即答。冒頭から前のめりな印象を与えない |

---

## ②アイスブレイク

| 項目 | 内容 |
|---|---|
| 目的 | 緊張をほぐし、率直に話しやすい関係を作る |
| 時間配分 | 3〜5分（60分想定）／1〜2分・省略可（30分想定） |
| 確認事項 | 相手企業の業種特有の状況（店舗数、繁忙期等）への軽い言及 |
| 話すポイント | 外食業全体の状況、共通の話題（人手不足、特定技能制度の広がり等） |
| 避ける話題 | 特定の競合他社への言及、政治的・センシティブな話題 |

---

## ③会社理解

| 項目 | 内容 |
|---|---|
| 目的 | 相手企業の基本情報・体制を把握し、以降の説明を相手に合わせて調整する |
| 時間配分 | 5分（60分想定）／2〜3分（30分想定） |
| 確認事項 | 業種・店舗数、特定技能1号人材の在籍人数感、在留期限管理の現状（Excel／システム／未管理） |
| 話すポイント | 「差し支えない範囲で、現状の管理方法を教えていただけますか」等、相手に話してもらう問いかけを中心にする |
| 避ける話題 | 詳細な個人情報を根掘り葉掘り聞くこと。この段階では概算の把握で十分 |

---

## ④課題確認

| 項目 | 内容 |
|---|---|
| 目的 | Slide2の内容を軸に、相手固有の課題感を引き出す |
| 時間配分 | 5分（60分想定）／3分（30分想定） |
| 確認事項 | 在留期限管理の一元化状況、N3・技能試験・実務経験の管理状況、担当者の属人化リスク |
| 話すポイント | 「誰が2号候補か見えていますか」という問いかけを中心に、相手の回答を引き出しながら共感を示す |
| 避ける話題 | 「御社は管理が甘い」といった否定的な評価。あくまで業界共通の課題として扱う |

---

## ⑤PowerPoint説明

| 項目 | 内容 |
|---|---|
| 目的 | Slide1〜8を通じて、共同検証という考え方とツールの中身を伝える |
| 時間配分 | 15〜20分（60分想定）／10〜12分（30分想定） |
| 確認事項 | 相手の反応（前のめりか、慎重か）を見ながら説明のペースを調整する |
| 話すポイント | `docs/sales/tokutei_navi_presenter_notes.md` の各スライドの要点に沿って進行する。特にSlide3（共同検証の考え方）とSlide6（成果物）は丁寧に扱う |
| 避ける話題 | 機能の詳細説明に時間をかけすぎること、契約・価格の話を前倒しすること |

---

## ⑥ヒアリング

| 項目 | 内容 |
|---|---|
| 目的 | `docs/hearing/company_hearing_sheet.md` の項目を用い、共同検証の対象を具体化する材料を集める |
| 時間配分 | 5〜10分（60分想定）／3〜5分（30分想定） |
| 確認事項 | 特定技能1号人材の人数感、在留期限別の分布、N3取得状況、共同検証への協力可否 |
| 話すポイント | 「概算で構いません」と伝え、回答のハードルを下げる |
| 避ける話題 | 正確な数値をこの場で強く求めること。即答を無理強いしない |

---

## ⑦共同検証の提案

| 項目 | 内容 |
|---|---|
| 目的 | Slide3・Slide6の内容を踏まえ、具体的な次のステップとして共同検証を提案する |
| 時間配分 | 5分（60分想定）／3分（30分想定） |
| 確認事項 | 対象人数感（5〜20名程度）、対象者選定の目処、社内確認が必要な事項の有無 |
| 話すポイント | 「まずは5〜20名程度で、一緒に棚卸ししてみませんか」という誘いの言葉。初期診断であり保証しない旨を改めて確認する |
| 避ける話題 | 「今日ご返事をください」といった即答を迫る表現 |

---

## ⑧クロージング

| 項目 | 内容 |
|---|---|
| 目的 | 面談全体を前向きに締めくくり、次回への合意形成をする |
| 時間配分 | 3〜5分（60分想定）／2分（30分想定） |
| 確認事項 | 本日話した内容の要点確認、次回の窓口担当者 |
| 話すポイント | 「今日は決断の場ではなかった」ことの再確認、感謝の言葉 |
| 避ける話題 | 「では契約書を用意します」等の先走った表現 |

---

## ⑨次回アクション

| 項目 | 内容 |
|---|---|
| 目的 | 具体的な持ち帰り事項と次回日程を明確にする |
| 時間配分 | 2〜3分（60分想定）／2分（30分想定） |
| 確認事項 | ヒアリングシートの記入担当者・期限、次回面談候補日 |
| 話すポイント | 「ヒアリングシートにご記入いただき、対象者の人数感が分かれば、共同検証の進め方をご相談できればと思います」と、具体的な行動として伝える |
| 避ける話題 | 曖昧なまま終わらせること。「またご連絡します」で終わらせず、具体的な次のアクションを確認して締める |

---

## 進行上の共通ルール

- 各フェーズの時間配分はあくまで目安。相手の反応・関心度に応じて柔軟に調整する
- 「共同検証」「共同棚卸し」という言葉を一貫して使い、「PoC」という言葉は概念説明（Slide3）以外では多用しない
- 制度判断が必要な場面では、その場で判断せず「専門家確認」へ接続する（`docs/sales/tokutei_navi_expected_questions.md` の共通ルールを参照）
- 面談全体を通じて、「今日は決断の場ではない」という前提を崩さない。急かす表現・クロージングを迫る表現は避ける
